zana.java contains small amounts of Java code for tasks where it's easier to create a reliable, high performance implementation in Java than in Clojure.

The most common reason is the type-hinting mess in Clojure. The syntax is inconsistent. Many type hints are ignored. There is support for double and long, but no other primitives.

Another reason is that inheritance is actually the right thing to do sometimes, cersus taking the risk of introducing inconsistencies thru cut-and-paste. @author John Alan McDonald @version 2015-11-27
